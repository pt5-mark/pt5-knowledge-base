📈 **项目热度表现**

- **微舆（BettaFish）**

  - 定位：多Agent舆情分析助手

  - GitHub星标：26.7k（截至2025年11月），曾空降GitHub Trending首位，星标增长曲线陡峭（2025年10月后快速突破25k）

  - 核心目标：打破信息茧房，还原舆情原貌，支持专业舆情风险预警与分众观点洞察

**TrendRadar**

定位：AI热点助手（Smart News Aggregator）

GitHub星标：11.6k（截至2025年11月），曾获GitHub Trending日榜第1，星标增速陡峭（2025年11月达10k+）

核心目标：告别无效刷屏，聚焦用户真正关心的新闻资讯，支持30秒快速部署

🔍 **核心功能对比**

| 维度                | 微舆（BettaFish）                          | TrendRadar                          |

|———————|——————————————|————————————|

| **核心能力**        | 专业舆情分析（品牌公关、投资决策、政府媒体） | 日常热点追踪（个人用户、内容创作者） |

| **数据覆盖**        | 国内外30+主流社媒，7+中文平台深度爬取      | 默认11个平台（知乎/抖音/百度热搜等），支持自定义添加 |

| **技术架构**        | 4大Agent协作（Insight/Media/Query/Report） | MCP分析框架，13种AI分析工具         |

| **部署难度**        | 轻量化Python模块化部署                     | 零门槛（GitHub Fork/30秒部署）      |

| **推送方式**        | 智能报告生成（HTML模板）                   | 多渠道推送（企业微信/飞书/Email等）  |

💡 **关键技术洞察**

- **微舆的多Agent协作机制**

  - 4个独立Agent通过ForumEngine实现“论坛协作”，避免单一模型局限：

    - Insight Agent：私有数据库挖掘

    - Media Agent：多模态内容分析（短视频/图片）

    - Query Agent：全球网页搜索验证

    - Report Agent：结构化报告生成（含品牌声量趋势图、情感分析）

  - 分析流程：用户提问→多Agent并行调研→论坛协作辩论→动态报告生成（支持210万文本样本处理）

**TrendRadar的精准筛选与趋势追踪**

关键词语法：普通词/必须词(+)/过滤词(!)，支持词组化管理（如“+AI !技术”）

热度分析工具：时间轴追踪（记录热点完整生命周期）、频次统计（不同时段排名变化）

推送模式：即时推送（新热点）/定时简报（日报/周报）/智能汇总（去重聚类）

📊 **适用场景与用户反馈**

- **微舆**：适合专业场景（舆情风险预警、分众观点洞察），但存在“幻觉问题”（部分分析结果不准确）和成本较高反馈

- **TrendRadar**：适合个人高效获取热点（如股市监控、行业动态），用户称赞其“轻量化”和“推送精准度”

🎯 **选择指南**

- 若需**日常热点追踪/快速过滤无效信息** → TrendRadar（轻量高效）

- 若需**专业舆情分析/深度观点洞察** → 微舆（BettaFish，专业全面）
