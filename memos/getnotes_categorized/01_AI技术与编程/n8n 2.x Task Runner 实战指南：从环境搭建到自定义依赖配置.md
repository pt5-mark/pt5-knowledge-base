### **📋 项目背景与目标**
**核心需求**：通过n8n实现自动化工作流：飞书接收视频链接 → 下载视频 → 提取字幕 → 飞书通知。

**技术基础**：n8n 2.x版本的**Task Runner**功能，其核心改进是将执行环境从主线程分离，支持Python代码运行及外部依赖扩展。

### **🔄 环境选择：从手机到Docker的迁移**
**(一) 手机端运行的局限性**

- **性能瓶颈**：手机息屏后CPU性能断崖式下降，任务持续转圈。

- **环境不稳定**：后台进程受限、电量管理导致执行中断。

**(二) Docker方案优势**

- **低试错成本**：Windows + Docker Desktop环境变量可控。

- **部署便捷**：通过docker-compose实现服务编排，支持数据卷挂载。

### **🚀 官方Task Runner部署与验证**
**(一) 部署步骤**

- **架构理解**：n8n负责调度，Runner负责执行，task broker作为通信桥梁。

**配置要点**：

- 使用SQLite数据库减少复杂度。

- 采用官方推荐的docker-compose范式。

- 通过AI和网络博文获取实操步骤。

### **🔧 进阶配置：自定义依赖与系统工具**
**(一) 官方Runner镜像限制**

- 系统软件管理工具（如apk）被删除，无法直接安装依赖。

- 需通过**多阶段构建**扩展镜像功能。

**(二) 自定义Dockerfile构建**

- **基础镜像**：基于python:3.13-alpine构建。

**关键步骤**：

- 从官方runners镜像拷贝核心程序（如task-runner-launcher）。

- 安装系统依赖：ffmpeg（视频处理）、ca-certificates（证书支持）。

- 安装Python库：yt-dlp（视频下载）、numpy、pandas等。

**(三) 依赖安装代码片段**
# 安装系统依赖
RUN apk add --no-cache ffmpeg libstdc++

# 安装Python依赖（使用清华源加速）
RUN uv pip install --no-cache-dir --index-url https://pypi.tuna.tsinghua.edu.cn/simple yt-dlp requests beautifulsoup4

### **⚠️ Allowlist配置：解决依赖调用限制**
**(一) 默认限制**
Task Runner启用严格的包运行列表（allowlist），默认禁止所有外部依赖，需手动配置允许列表。

**(二) 解决方案**

**修改配置文件**：下载官方n8n-task-runners.json，添加环境变量支持：

"allowed-env": [
 "N8N_RUNNERS_STDLIB_ALLOW",
 "N8N_RUNNERS_EXTERNAL_ALLOW"
]

**容器启动时配置**：在docker-compose.yml中通过环境变量开放权限：
“`yaml
environment:

- N8N_RUNNERS_STDLIB_ALLOW=*  # 允许所有标准库

N8N_RUNNERS_EXTERNAL_ALLOW=*  # 允许所有外部库
“`
