**为什么会出现：**

1. **子代理并发/频率过高** — 你启动了一些后台子任务（可能是 codex、自动化任务或者我这边调用的 sub-agent），它们短时间内频繁调用模型 API
2. **模型提供商的限制** — kimi-coding/k2p5 或其他模型有 RPM（每分钟请求数）限制，子代理没做退避重试

**解决方案：**

```bash
# 查看当前运行的子代理
openclaw sessions list

# 如果有不需要的，清理掉
openclaw sessions kill <session-id>
```

或者在你的配置里加 rate limiting：

```json
// ~/.openclaw/openclaw.json
{
  "agents": {
    "defaults": {
      "rateLimit": {
        "requestsPerMinute": 30
      }
    }
  }
}
```
